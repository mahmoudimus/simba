# Simba Recall Levers Implementation Spec

## C3 — True LLM HyDE (`memory.hyde_mode: "keyword" | "llm"`)

### Goal

Replace the current focus-term string that seeds the second vector arm with a short hypothetical answer generated by the LLM. This gives the second arm genuine semantic content rather than a bag of keywords. When `hyde_mode = "llm"` and generation fails for any reason, the arm collapses to `""` (no extra embedding), preserving current behavior. The daemon hot path is never blocked: a `HydeCache` mirrors `RerankCache` exactly (LRU, `signature`/`get`/`put`), so the first recall serves the keyword fallback and populates the cache off the hot path; subsequent identical queries get the HyDE text free.

```python
import collections, hashlib, re

class HydeCache:
    def __init__(self, max_entries: int = 256) -> None: ...

    def signature(self, query: str) -> str:
```

## IRCoT — Answer-Time Multi-Hop QA

### Goal

An interleaved retrieve-and-reason loop that runs at benchmark judge time (not at daemon recall time). Given a multi-hop question, IRCoT generates a reasoning step and a follow-up sub-query via the LLM, retrieves for it, appends the evidence, and repeats up to N steps before generating a final answer. This is a pure, injectable module — the retriever and LLM are always injected so the entire loop is exercised with fakes in CI. It replaces `score_case` for multi-hop questions in `run_qa` when `ircot_enabled=True`.

```python
from __future__ import annotations
import typing

if typing.TYPE_CHECKING:
    from simba.eval.dataset import EvalCase

Retriever = typing.Callable[[str], list[str]]


def build_step_prompt(
    question: str,
    evidence: list[str],
    step: int,
) -> str:
    """Prompt that asks the LLM for one reasoning sentence and a follow-up sub-query.

    Returns a prompt expecting JSON: {"reasoning": "...", "sub_query": "..."}.
    ``evidence`` is the accumulated evidence so far (may be empty on step 0).
    """


def build_final_prompt(question: str, evidence: list[str]) -> str:
    """Prompt that asks the LLM to answer the question from accumulated evidence.

    Structurally identical to judge.build_answer_prompt — reuses it.
    """


def ircot_answer(
    question: str,
    retriever: Retriever,
    id2content: dict[str, str],
    llm: typing.Any,
    *,
    max_steps: int = 4,
    k_per_step: int = 3,
    k_final: int = 10,
) -> str:
    """Run the IRCoT loop and return a final answer string (or "" on failure).

    Algorithm:
      evidence_ids: list[str] = []
      for step in range(max_steps):
          evidence_texts = [id2content[i] for i in evidence_ids[-k_final:] if i in id2content]
          step_json = llm.complete_json(build_step_prompt(question, evidence_texts, step))
          if step_json is None or not step_json.get("sub_query"):
              break
          sub_query = step_json["sub_query"]
          new_ids = retriever(sub_query)[:k_per_step]
          evidence_ids.extend(i for i in new_ids if i not in set(evidence_ids))
      final_evidence = [id2content[i] for i in evidence_ids[:k_final] if i in id2content]
      return llm.complete(build_final_prompt(question, final_evidence))

    Fail-open: any exception → return "".
    """


def score_case_ircot(
    case: EvalCase,
    retriever: Retriever,
    id2content: dict[str, str],
    llm: typing.Any,
    *,
    max_steps: int = 4,
    k_per_step: int = 3,
    k_final: int = 10,
    cache: typing.Any = None,
    judge_model: str = "",
    judge_llm: typing.Any | None = None,
) -> bool | None:
    """IRCoT version of judge.score_case: same return contract (True/False/None).

    Runs ircot_answer to produce the predicted answer, then grades it with
    ``judge_llm`` (or ``llm`` if ``judge_llm`` is None) using judge.build_judge_prompt.
    ``cache`` is the same JudgeCache as in score_case.
    """
```

```python
    base = simba.config.load("eval")
    valid = {f.name for f in dataclasses.fields(EvalConfig)}
    filtered = {k: v for k, v in overrides.items() if v is not None and k in valid}

    merged = dataclasses.asdict(base)

    return EvalConfig(**merged)
```

```python
class FakeLlm:
    """Tracks calls; returns configured step JSON and final answer."""
    def __init__(
        self,
        step_responses: list[dict],
        final_answer: str = "The answer",
        verdict: dict | None = None,
    ) -> None:
        self.step_responses = list(step_responses)
        self.final_answer = final_answer
        self.verdict = verdict or {"correct": True}
        self.complete_calls: list[str] = []
        self.complete_json_calls: list[str] = []

    def complete(self, prompt: str) -> str:
        self.complete_calls.append(prompt)
        return self.final_answer

    def complete_json(self, prompt: str) -> object:
        self.complete_json_calls.append(prompt)
        if self.step_responses:
            return self.step_responses.pop(0)
        return self.verdict  # judge call


def _make_retriever(mapping: dict[str, list[str]]) -> Retriever:
    """Returns a retriever that maps sub-queries to fixed id lists."""
    return lambda q: mapping.get(q, [])
```

### Acceptance

- `simba config set eval.ircot_enabled true` → `run_qa` routes multi-hop cases through `ircot_answer`.
- `simba config set eval.ircot_enabled false` → identical behavior to today.
- The loop is pure/injectable: no LanceDB, no LLM subprocess in tests.
- The IRCoT path measures QA accuracy (not recall@k), so the reported metric is `overall.accuracy` from `aggregate(rows)`, directly comparable between `ircot_enabled=true` and `ircot_enabled=false` on the same dataset.
- The multi-hop QA accuracy delta is the primary measurement target.
